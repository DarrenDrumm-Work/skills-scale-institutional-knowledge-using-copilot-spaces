## Purpose
Provide a standardized checklist for managing change requests throughout the project lifecycle. This checklist helps ensure all changes are properly evaluated, approved, and communicated.

## Related Roles
- [Change Manager](./octoacme-roles-and-personas.md#change-manager)
- [Project Manager](./octoacme-roles-and-personas.md#project-managers)

---

## Change Request Submission
- [ ] Change request form completed with description and justification
- [ ] Requestor and date identified
- [ ] Category assigned (Scope, Schedule, Resource, Technical, Other)
- [ ] Priority level indicated (Critical, High, Medium, Low)

## Impact Assessment
- [ ] Scope impact evaluated (features affected, new work required)
- [ ] Schedule impact assessed (timeline changes, dependencies)
- [ ] Resource impact analyzed (effort, skills, availability)
- [ ] Risk assessment completed (new risks introduced)
- [ ] Cost implications documented (if applicable)
- [ ] Technical feasibility confirmed with development team

## Stakeholder Review
- [ ] Affected stakeholders identified
- [ ] Stakeholder communication plan defined
- [ ] Review meeting scheduled (if required for significant changes)
- [ ] All required approvals obtained

## Approval & Documentation
- [ ] Change request approved/rejected decision documented
- [ ] Approval signatures or electronic confirmations recorded
- [ ] Change log updated with decision and rationale
- [ ] Related project documents updated (scope, schedule, risk register)

## Communication & Implementation
- [ ] Change notification sent to affected parties
- [ ] Implementation timeline communicated
- [ ] Resource assignments updated (if needed)
- [ ] Dependencies and blockers identified

## Post-Implementation
- [ ] Change implementation verified
- [ ] Lessons learned captured (for significant changes)
- [ ] Change log closed with final status
- [ ] Stakeholder signoff obtained

## Change Categories
- **Scope**: Changes to features, requirements, or deliverables
- **Schedule**: Changes to timelines, milestones, or deadlines
- **Resource**: Changes to team composition, effort, or skills
- **Technical**: Changes to architecture, technology, or implementation approach
- **Other**: Regulatory, compliance, or external dependencies

## Acceptance Criteria
- [ ] This checklist is used for all change requests
- [ ] Change log is updated weekly or as changes occur
- [ ] Stakeholders are notified of approved changes within 24 hours
