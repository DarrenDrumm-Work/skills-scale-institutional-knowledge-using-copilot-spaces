# OctoAcme — Communications Plan Template

## Purpose
Provide a standardized template for developing and maintaining project communications plans. This template helps ensure consistent, effective stakeholder communication throughout the project lifecycle.

## Communication Objectives
- Ensure all stakeholders receive timely, relevant project updates
- Maintain transparency on project status, risks, and decisions
- Support change adoption through effective messaging
- Build stakeholder confidence and engagement

## Stakeholder Communication Matrix

| Stakeholder Group | Key Contacts | Information Needs | Frequency | Channel | Owner | Format |
|-------------------|--------------|-------------------|-----------|---------|-------|--------|
| Executive Sponsors | Names | High-level status, decisions needed | Monthly | Email, Meeting | PM | Executive Summary |
| Project Team | Team leads | Detailed progress, blockers, tasks | Daily/Weekly | Standup, Slack | PM | Verbal, Board |
| Business Stakeholders | Names | Feature updates, timelines | Bi-weekly | Email, Newsletter | Comms Lead | Status Update |
| End Users | Group/Channel | Release notes, training | Per release | Email, Wiki | Comms Lead | Announcement |
| Support/Operations | Names | Deployment info, known issues | Per release | Email, Runbook | PM | Release Notes |

## Communication Types

### Regular Communications
| Type | Purpose | Frequency | Owner | Template |
|------|---------|-----------|-------|----------|
| Weekly Status Update | Progress, risks, next steps | Weekly | PM | [Status Template](./octoacme-risks-and-communication.md#communication-templates) |
| Team Standup | Daily sync, blockers | Daily | Scrum Master/PM | Verbal agenda |
| Stakeholder Newsletter | Project highlights, upcoming work | Bi-weekly/Monthly | Comms Lead | Newsletter format |
| Demo/Review | Show completed work | End of sprint | PM/Tech Lead | Demo agenda |

### Event-Driven Communications
| Type | Trigger | Audience | Owner | Template |
|------|---------|----------|-------|----------|
| Release Announcement | Production deployment | All stakeholders | Comms Lead | [Release Notes](./octoacme-release-and-deployment.md#release-notes-template) |
| Change Notification | Approved change | Affected stakeholders | Change Manager | Change notice |
| Incident Communication | Production issue | Affected users, support | PM/On-call | [Incident Template](./octoacme-risks-and-communication.md#communication-templates) |
| Risk Escalation | High-impact risk identified | PM, Sponsors | PM | Escalation notice |

## Communication Channels

| Channel | Best For | Guidelines |
|---------|----------|------------|
| Email | Formal updates, decisions, approvals | Use clear subject lines, include action items |
| Slack/Teams | Quick questions, informal updates | Use threads, respect channel purpose |
| Meetings | Complex discussions, alignment | Send agenda beforehand, share notes after |
| Project Wiki/Docs | Reference information, documentation | Keep updated, single source of truth |
| Project Board | Task status, workflow | Keep current, use consistent labels |

---

## Escalation Path
For communication issues or gaps:
1. Team level → Communications Lead
2. Communications Lead → Project Manager
3. Project Manager → Project Sponsor
